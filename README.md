# ghazalkarami-beauty

اینجا خانه‌ی امن زیبایی‌ست

سایت سالن تخصصی مو و آکادمی غزل کرمی (به همراه دنیا سلیمانی) — فرانت‌اند طراحی‌شده به‌علاوه‌ی بک‌اند واقعی برای رزرو نوبت، احراز هویت، پرداخت و پنل مدیریت.

## ساختار پروژه

```
public/              فرانت‌اند سایت (همون طراحی اصلی، حالا متصل به بک‌اند واقعی)
admin-panel/          پنل مدیریت غزل (ساعات کاری، قیمت‌ها، گالری، نظرات، رزروها)
backend/               API با Node.js + Express + TypeScript + Prisma + PostgreSQL
frontend-reference/   نسخه‌ی اصلی HTML آپلودشده، به‌عنوان مرجع/بکاپ (بدون تغییر)
```

## راه‌اندازی برای توسعه

```bash
cd backend
cp .env.example .env    # مقادیر رو با اطلاعات واقعی پر کنید
npm install
npx prisma migrate dev
npm run prisma:seed
npm run dev              # سرور روی http://localhost:4000 بالا میاد
```

سایت اصلی: `http://localhost:4000/`
پنل مدیریت: `http://localhost:4000/admin/`

اطلاعات ورود پیش‌فرض پنل مدیریت از `ADMIN_SEED_PHONE` / `ADMIN_SEED_PASSWORD` در فایل `.env` ساخته می‌شه.

## چیزهایی که قبل از رفتن به حالت واقعی (production) باید پر بشه

فایل `backend/.env` را ببینید:

- `DATABASE_URL` — آدرس دیتابیس PostgreSQL واقعی
- `JWT_SECRET` — یک رشته‌ی تصادفی و طولانی
- `KAVENEGAR_API_KEY` و ۴ تا `KAVENEGAR_TEMPLATE_*` — این‌ها باید اول به‌صورت «پترن» تاییدشده در پنل کاوه‌نگار ساخته بشن (پیامک‌های خدماتی با متن آزاد قابل ارسال نیستن)؛ توضیح کامل توکن‌های هر پترن در `.env.example` هست
- `ZARINPAL_MERCHANT_ID` — برای اتصال واقعی به درگاه پرداخت (تا وقتی خالیه، سیستم در «حالت آزمایشی» پرداخت رو خودکار موفق در نظر می‌گیره تا بشه کل جریان رزرو رو تست کرد)
- `ADMIN_SEED_PHONE` / `ADMIN_SEED_PASSWORD` — شماره و رمز عبور غزل برای ورود به پنل مدیریت

بدون این مقادیر، سایت کامل کار می‌کند اما پیامک واقعی ارسال نمی‌شود و پرداخت واقعاً به زرین‌پال وصل نمی‌شود (فقط لاگ می‌شود / شبیه‌سازی می‌شود).

## نکات مهم درباره‌ی زمان‌بندی رزرو

- سرویس‌های **پروتئین‌تراپی، کراتینه، بوتاکس، کرابوتاکس، رنگ‌ولایت و دکلره‌ی پیکسی** یک خط زمانی مشترک دارن (دسته‌بندی `chem` در دیتابیس) — رزرو هرکدوم (چه آنلاین چه دستی در پنل) همون تایم رو برای بقیه‌ی این گروه هم می‌بنده.
- **هیرکات** و **اسکراب اسکالپ** هرکدوم خط زمانی مستقل خودشون رو دارن و از این قفل مشترک تاثیر نمی‌گیرن.
- «رنگ و لایت» چون فقط با هماهنگی در دایرکت اینستاگرام رزرو می‌شه، در سایت گزینه‌ی رزرو آنلاین نداره؛ وقتی غزل از پنل مدیریت (تب «رزروها») یک رزرو دستی برای رنگ‌ولایت ثبت می‌کنه، همون تایم برای بقیه‌ی گروه هم‌خط هم قفل می‌شه.
- روزهای کاری، استثناهای تاریخ خاص (باز کردن یه شنبه/جمعه‌ی خاص) و تعطیلی موقت (مرخصی) همگی از تب «زمان‌بندی سالن» در پنل مدیریت قابل تنظیمن.
